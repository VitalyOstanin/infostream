## 1. Обзор тарифа Team

Claude Team Plan - платный командный тариф для работы с Claude через веб-интерфейс claude.ai. Предназначен для команд от 5 до 150 человек.

Что входит:
- Доступ ко всем моделям Claude (включая последние)
- Доступ к Claude Code
- Проекты с общими базами знаний
- Контекстное окно до 1M токенов для Opus 4.6 и Sonnet 4.6 (с 13 марта 2026, ранее было 200k)
- Коннекторы: Google Drive, Gmail, Calendar, GitHub, Microsoft 365, Slack
- SSO, Domain Capture, JIT-provisioning (SCIM-provisioning доступен только на Enterprise)
- Централизованное управление: роли, биллинг, аналитика использования
- Контроль расходов на уровне организации и отдельных пользователей
- Enterprise Search с предварительно настроенными инструкциями

**Не входит в Team (только Enterprise):** SCIM-provisioning, audit logs, настройка срока хранения данных (data retention), приоритетная поддержка

## 2. Требования для регистрации

- **Рабочая email-почта** - публичные домены (@gmail.com, @yahoo.com и т.п.) **не допускаются**. Нужен корпоративный домен.
- **Минимум 5 участников** - нельзя создать Team-план менее чем на 5 мест.
- Все участники должны иметь email на разрешённых корпоративных доменах организации.

## 3. Стоимость

| Тип места | Помесячная оплата | Годовая оплата (за месяц) |
|-----------|-------------------|---------------------------|
| Standard  | $25/мес за человека | $20/мес за человека ($240/год) |
| Premium   | $125/мес за человека | $100/мес за человека ($1200/год) |

Пример для команды из 5 человек (3 Standard + 2 Premium):

| | Помесячная оплата | Годовая оплата |
|---|---|---|
| 3 Standard | 3 x $25 = **$75/мес** | 3 x $20 = **$60/мес** ($720/год) |
| 2 Premium | 2 x $125 = **$250/мес** | 2 x $100 = **$200/мес** ($2400/год) |
| **Итого** | **$325/мес** | **$260/мес** ($3120/год, оплата вперёд за весь год) |

Цены указаны без учёта налогов (VAT/НДС), которые зависят от региона.

## 4. Создание организации и подключение Team-тарифа

### 4.1. Новая регистрация

1. Откройте [claude.ai/login](https://claude.ai/login).
2. Введите **рабочий email** (корпоративный домен, не gmail/yahoo).
3. Пройдите онбординг-процесс (подтверждение email, создание аккаунта).
4. Выберите **Team plan** среди предложенных тарифов.
5. Укажите название организации.
6. Выберите тип биллинга: **Monthly** (помесячно) или **Annual** (годовой).
7. Введите данные платёжной карты.
8. Укажите начальное количество мест (минимум 5).
9. Подтвердите покупку.

### 4.2. Апгрейд с существующего аккаунта

Если у вас уже есть аккаунт Claude (Free, Pro или Max):

1. Войдите в свой существующий аккаунт на [claude.ai](https://claude.ai).
2. Перейдите на [claude.ai/upgrade](https://claude.ai/upgrade).
3. Следуйте шагам по созданию Team-организации.

**Важно:**
- Будет создана **отдельная организация** - она не заменяет ваш личный аккаунт.
- Личный аккаунт (Free/Pro/Max) остаётся независимым.
- Переключение между аккаунтами - через инициалы/имя в нижнем левом углу интерфейса.
- Данные между личным и командным аккаунтом **не переносятся**.
- Если личный платный аккаунт больше не нужен, его подписку нужно **отменить отдельно** в настройках этого аккаунта.

## 5. Настройка оплаты

Оплата производится **банковской картой** (кредитная, дебетовая или предоплаченная). Другие способы оплаты (ACH, банковский перевод) **не поддерживаются**. Управление биллингом доступно только ролям **Owner** и **Primary Owner**.

Путь: **Organization settings > Billing**.

Здесь можно:
- Просматривать и оплачивать счета
- Добавлять/менять платёжные методы
- Добавить Tax ID / VAT ID организации (для корректного налогообложения)
- Переключаться между помесячной и годовой оплатой

Для добавления VAT ID: Organization settings > Billing > Tax Information.

Счета автоматически приходят на email и доступны для скачивания через Stripe в разделе Billing. Тема письма: "Your receipt from Anthropic."

Для изменения email для биллинга нужно обращаться в поддержку Anthropic с подтверждением от Owner.

Освобождение от налогов (tax-exempt) нельзя настроить при регистрации. Для этого нужно после подписки отправить подтверждающие документы в поддержку.

## 6. Покупка и управление местами (seats)

### 6.1. Типы мест

- **Standard** - базовые возможности, лимит использования 1.25x от Pro-тарифа за 5-часовую сессию. Включает Claude Code.
- **Premium** - всё из Standard + значительно больший лимит использования (6.25x от Pro). Для активных пользователей.

Можно **комбинировать** типы мест: например, 2 Premium для активных пользователей (разработчики, аналитики) и 3 Standard для остальных участников команды.

### 6.2. Покупка дополнительных мест

1. Войдите с ролью **Owner** или **Primary Owner**.
2. Перейдите в **Organization settings > Organization**.
3. Нажмите **Manage** в разделе "Total seats".
4. В появившемся окне выберите **Add or change seats**.
5. Нажмите **"+"** рядом с нужным типом места (Standard или Premium).
6. Просмотрите влияние на биллинг и подтвердите покупку.

Места также можно купить прямо при добавлении нового участника.

### 6.3. Уменьшение количества мест

1. Сначала удалите или переназначьте участников, чтобы освободить места.
2. Откройте интерфейс управления местами (Organization settings > Organization > Manage).
3. Нажмите **"-"** для уменьшения количества мест нужного типа.
4. Подтвердите изменения.

### 6.4. Смена типа места у сотрудника

Роли Admin и выше могут переключать участника между Standard и Premium:

1. Перейдите в **Organization settings > Organization**.
2. Найдите нужного участника в списке.
3. Нажмите на выпадающий список в колонке **Seat Tier**.
4. Выберите новый тип места.

Это переназначение в рамках существующих мест. Если свободных мест Premium нет, сначала нужно их докупить.

## 7. Приглашение сотрудников

### 7.1. По email

1. Перейдите в **Organization settings > Organization**.
2. Нажмите кнопку приглашения участников.
3. Введите email-адреса (можно несколько через запятую или с новой строки).
4. Email должен соответствовать разрешённым доменам организации.
5. Отправьте приглашения.

**Важно:** приглашение **сразу занимает место** (seat), даже если сотрудник ещё не принял приглашение.

### 7.2. По ссылке-приглашению

На Team-плане ссылки-приглашения включены по умолчанию. Генерировать и управлять ссылками могут роли **Admin** и выше.

1. Перейдите в **Organization settings > Identity and access**.
2. Сгенерируйте или скопируйте ссылку-приглашение.
3. Отправьте ссылку коллегам (через Slack, email, внутренний wiki и т.п.).
4. При переходе по ссылке сотрудник присоединяется к организации **автоматически** (без процесса одобрения).
5. Ему назначается **самый базовый доступный тип места** (Standard). Если свободных мест нет - автоматически создаётся новое Standard-место.
6. При необходимости администратор может потом изменить тип места на Premium.

**Ограничения:**
- Ссылка действует **90 дней** с момента создания, после чего нужно сгенерировать новую.
- Работает ограничение по доменам email (присоединиться могут только пользователи с разрешённых доменов).
- При достижении лимита мест (если авто-создание места невозможно) пользователь получит ошибку.

### 7.3. Автообнаружение организации

Когда функция включена, коллеги с email на вашем корпоративном домене могут самостоятельно найти организацию и отправить запрос на вступление.

## 8. Роли и права доступа

В Team-плане доступны 4 роли (от минимальных прав к максимальным):

| Роль | Права |
|------|-------|
| **User** | Создание чатов, использование проектов, запрос экспорта данных |
| **Admin** | Всё из User + приглашение/удаление участников, смена типов мест, управление ссылками-приглашениями |
| **Owner** | Всё из Admin + управление ролями, включение интеграций и возможностей, управление публичными проектами, просмотр аналитики, просмотр/оплата счетов, управление платёжными методами |
| **Primary Owner** | Всё из Owner + покупка/уменьшение мест, передача роли Primary Owner, запрос экспорта данных организации |

**Primary Owner может быть только один** в организации. Он занимает одно из оплаченных мест.

Роль назначается при приглашении участника и может быть изменена позднее владельцами (Owner/Primary Owner).

## 9. Удаление участников

1. Перейдите в **Organization settings > Organization**.
2. Найдите участника в списке.
3. Нажмите на меню (три точки) рядом с его именем.
4. Выберите удаление.

Участник **теряет доступ немедленно**. Его место освобождается для переназначения. Если позже добавить этого человека по тому же email - его история сохранится.

**Важно:** удаление участника **не уменьшает** количество оплаченных мест. Место освобождается и может быть назначено другому сотруднику. Чтобы перестать платить за это место, нужно дополнительно **уменьшить количество мест** (см. раздел 6.3). При уменьшении количества мест начисляется пропорциональный кредит.

## 10. Особенности биллинга

- **Добавление мест** - оплата пропорционально оставшемуся времени в биллинговом периоде (prorated), списывается сразу.
- **Апгрейд места** (Standard -> Premium) - разница в цене пропорционально списывается сразу.
- **Удаление участника** - место остаётся оплаченным и доступно для переназначения другому сотруднику.
- **Уменьшение количества мест** - начисляется пропорциональный кредит на оставшееся время.
- **Начало периода** - оплата за все активные места в начале каждого биллингового цикла.

Пример расчёта для команды из 5 человек (помесячная оплата):

**Начальная конфигурация:** 3 Standard + 2 Premium = (3 x $25) + (2 x $125) = **$325/мес**

**Апгрейд 1 места с Standard до Premium в середине месяца** (16 дней до конца при 31-дневном месяце):
- Разница в цене: $125 - $25 = $100/мес
- Доплата: $100 x 1 место x 16 дней / 31 день = **$51.61** - списывается сразу
- Со следующего месяца: 2 Standard + 3 Premium = (2 x $25) + (3 x $125) = **$425/мес**

Для экспорта данных об участниках (имя, email, роль, тип места) доступен CSV-экспорт в разделе Members.
