## Описание проблемы

Claude Code использует **kitty keyboard protocol** (progressive keyboard enhancement)
для расширенной обработки клавиш. Этот протокол позволяет приложению получать
детальную информацию о нажатиях (различать Tab и Ctrl+I, Backspace и Ctrl+H и т.д.).

При выходе из Claude Code (особенно аварийном через Ctrl+C или при потере соединения)
терминал может остаться в "расширенном" режиме клавиатуры. Симптомы:

- Мусорные символы при обычном вводе в shell (например, `[1;5A` вместо стрелки вверх)
- Не работают привычные комбинации клавиш (Ctrl+C, Ctrl+D, Ctrl+R)
- Дублирование символов при вводе
- Автодополнение (Tab) выдает escape-последовательности вместо результатов

Параллельно существуют ещё две проблемы:

- Горячие клавиши Kitty (Ctrl+C, Ctrl+W, Alt+D) не работают при **русской раскладке**
- OSC-последовательности от shell integration могут "просачиваться" в вывод

## Требования

- Терминал Kitty (0.35+, с поддержкой kitty keyboard protocol)
- Claude Code
- Bash или Zsh

## Решение 1: Враппер-скрипт для запуска Claude Code

Идея: после завершения процесса `claude` выполнить набор escape-последовательностей,
которые гарантированно возвращают терминал в нормальное состояние.

### Скрипт `claude-code-work.sh`

```bash
#!/bin/bash

# Запуск Claude Code
claude "$@"

# === Сброс терминала после выхода из Claude Code ===

# 1. Pop все уровни kitty keyboard enhancement protocol
#    Kitty protocol использует стек: приложение делает push при входе,
#    pop при выходе. Если приложение не сделало pop — делаем принудительно.
printf '\x1b[<u'

# 2. Явный сброс: отключить все keyboard enhancement flags
#    Даже если pop не сработал (стек уже пуст), эта команда
#    устанавливает flags=0, полностью отключая расширенный режим.
printf '\x1b[=0u'

# 3. Сброс xterm modifyOtherKeys
#    Ещё один механизм расширенного ввода клавиш (не kitty-специфичный,
#    но Claude Code может его активировать). modifyOtherKeys=0 — отключить.
printf '\x1b[>0m'

# 4. Восстановить нормальные настройки терминального драйвера
#    Переводит tty из raw-режима обратно в cooked: восстанавливает
#    обработку Ctrl+C (SIGINT), echo, line editing и т.д.
stty sane 2>/dev/null
```

### Что делает каждая команда сброса

| N | Escape-последовательность | Протокол              | Действие                                                         |
|---|---------------------------|-----------------------|------------------------------------------------------------------|
| 1 | `\x1b[<u`                | Kitty keyboard        | Pop всех уровней стека keyboard enhancement                      |
| 2 | `\x1b[=0u`               | Kitty keyboard        | Принудительный сброс всех флагов (flags=0) — подстраховка для #1 |
| 3 | `\x1b[>0m`               | xterm modifyOtherKeys | Отключение расширенного режима модификаторов клавиш               |
| 4 | `stty sane`              | POSIX tty             | Восстановление нормального режима терминального драйвера          |

Команды 1 и 2 покрывают kitty keyboard protocol с двух сторон: через стек (pop)
и через прямую установку флагов. Команда 3 покрывает альтернативный протокол xterm.
Команда 4 — низкоуровневая подстраховка на случай, если терминальный драйвер
остался в raw-режиме.

### Установка и использование

```bash
# Скопировать скрипт
cp claude-code-work.sh ~/scripts/claude-code-work.sh
chmod +x ~/scripts/claude-code-work.sh

# Добавить алиас в ~/.bashrc или ~/.zshrc
alias cc='~/scripts/claude-code-work.sh'
```

После этого запускать Claude Code через `cc` вместо `claude`.

Если терминал уже сломался (Claude Code упал без нормального завершения),
можно выполнить сброс вручную — набрать вслепую и нажать Enter:

```bash
printf '\x1b[<u' && printf '\x1b[=0u' && printf '\x1b[>0m' && stty sane
```

## Решение 2: Горячие клавиши для русской раскладки

### Проблема

Kitty (как и большинство терминалов) привязывает горячие клавиши к физическим
кодам клавиш, основанным на латинской раскладке. При переключении на русскую
раскладку Ctrl+C отправляет не `\x03` (ETX/SIGINT), а `Ctrl+с` (кириллическая "с"),
который терминал не распознает как сигнал прерывания.

Аналогично перестают работать Ctrl+W (удалить слово), Alt+D (удалить слово вперед),
Ctrl+V (вставка) и другие привычные комбинации.

### Настройка kitty.conf

Добавить маппинг кириллических клавиш на соответствующие управляющие коды:

```conf
# Поддержка горячих клавиш при русской раскладке
map ctrl+shift+с copy_to_clipboard
map ctrl+shift+м paste_from_clipboard
map ctrl+с send_text all \x03
map ctrl+ц send_text all \x17
map alt+в send_text all \x1bd
```

Расшифровка:

| Маппинг              | Клавиша (рус.) | Эквивалент (лат.) | Действие                          |
|----------------------|----------------|--------------------|-----------------------------------|
| `ctrl+shift+с`      | Ctrl+Shift+С   | Ctrl+Shift+C       | Копирование в буфер обмена        |
| `ctrl+shift+м`      | Ctrl+Shift+М   | Ctrl+Shift+V       | Вставка из буфера обмена          |
| `ctrl+с`            | Ctrl+С         | Ctrl+C             | Отправка SIGINT (`\x03`)          |
| `ctrl+ц`            | Ctrl+Ц         | Ctrl+W             | Удаление слова назад (`\x17`)     |
| `alt+в`             | Alt+В          | Alt+D              | Удаление слова вперед (`\x1bd`)   |

Можно расширить список по необходимости, добавив другие часто используемые
комбинации (например, `ctrl+з` для Ctrl+Z — `\x1a`).

После изменения `kitty.conf` перезагрузить конфигурацию:

```bash
# Перезагрузка конфигурации без перезапуска kitty
kitty @ load-config
# или Ctrl+Shift+F5 в окне kitty
```

## Решение 3: Подавление утечки OSC-последовательностей

Shell integration в Kitty (и других терминалах) использует OSC-последовательности
для уведомлений о завершении команд. Claude Code генерирует много вывода,
и эти последовательности иногда "протекают" в видимый текст — появляются
артефакты вроде `]133;D;0` или `]633;E;`.

Решение — отключить уведомления о завершении команд в `kitty.conf`:

```conf
# Отключить уведомления о завершении команд (фикс утечки OSC-последовательностей)
notify_on_cmd_finish never
```

## Полный пример kitty.conf

Минимальный набор настроек, связанных с исправлением проблем при работе
с Claude Code:

```conf
# Отключить уведомления о завершении команд (фикс утечки OSC-последовательностей)
notify_on_cmd_finish never

# Поддержка горячих клавиш при русской раскладке
map ctrl+shift+с copy_to_clipboard
map ctrl+shift+м paste_from_clipboard
map ctrl+с send_text all \x03
map ctrl+ц send_text all \x17
map alt+в send_text all \x1bd
```

## Диагностика

### Проверить текущий режим клавиатуры

Если подозреваете, что терминал "застрял" в расширенном режиме:

```bash
# Показать текущие настройки tty
stty -a

# Если видите -echo, -icanon — терминал в raw-режиме
# Быстрый сброс:
stty sane
```

### Проверить, активен ли kitty keyboard protocol

```bash
# Запросить текущие флаги keyboard enhancement
printf '\x1b[?u'
# Ответ \x1b[?<flags>u — если flags != 0, протокол активен
```

### Ручной сброс при полностью сломанном вводе

Если ввод настолько сломан, что невозможно набрать команду:

1. Нажать `Ctrl+Shift+F5` — перезагрузить конфигурацию Kitty
2. Если не помогло — открыть новую вкладку (`Ctrl+Shift+T`) или новое окно
3. В крайнем случае — закрыть окно терминала и открыть новое
